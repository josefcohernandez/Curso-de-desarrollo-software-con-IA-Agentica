# Ejercicio: Plan de Adopción de IA para un Equipo

## Objetivo

Diseñar un plan de adopción de 4 fases para un equipo de 8 developers, incluyendo timeline, actividades concretas, métricas y criterios de avance entre fases.

**Tiempo estimado: 15 minutos**

---

## Escenario

Eres engineering manager de un equipo de 8 developers con estos perfiles:

| Developer | Experiencia | Actitud hacia IA |
|-----------|-------------|-------------------|
| Ana | Senior, 10 años | Entusiasta, ya usa Copilot por su cuenta |
| Carlos | Senior, 8 años | Escéptico, prefiere hacer todo manual |
| Diana | Mid, 4 años | Curiosa pero cautelosa |
| Elena | Mid, 5 años | Abierta a probar |
| Fernando | Junior, 1 año | Muy entusiasta, quiere usarlo para todo |
| Gabriel | Senior, 12 años | Resistente — "no confío en código que no escribí" |
| Hugo | Mid, 3 años | Indiferente |
| Isabel | Junior, 2 años | Nerviosa — "me va a quitar el trabajo" |

El equipo trabaja en un backend en Python (FastAPI) para una empresa de e-commerce. La empresa aprobó un presupuesto de $100/mes por developer para herramientas de IA.

---

## Instrucciones

### Parte 1: Plan por Fases

Para cada una de las 4 fases de adopción, define:

1. **Duración**: cuántas semanas
2. **Participantes**: quiénes participan en esta fase
3. **Actividades**: qué se hace concretamente
4. **Métricas**: qué se mide para saber si funciona
5. **Criterio de avance**: qué debe cumplirse para pasar a la siguiente fase

### Parte 2: Gestión de Resistencia

Para Carlos, Gabriel e Isabel, diseña una estrategia específica:

- ¿Qué tipo de resistencia tiene cada uno? (ver [02-resistencia-cambio.md](../teoria/02-resistencia-cambio.md))
- ¿Qué acción concreta tomarías con cada uno?
- ¿En qué fase del plan esperarías que cada uno se sume voluntariamente?

### Parte 3: Riesgos

Identifica 3 riesgos del plan y define una mitigación para cada uno.

---

```markdown
## Plan de Adopción — Equipo E-commerce Backend

### Fase 1: Exploración (Semana X-Y)

- **Criterio de avance**: ...

...

### Fase 3: Integración (Semana X-Y)
...

### Fase 4: Optimización (Semana X+)
...

### Gestión de Resistencia
- **Carlos**: ...
- **Gabriel**: ...
- **Isabel**: ...

### Riesgos y Mitigaciones
1. ...
2. ...
3. ...
```

---

## Criterios de Éxito

```markdown
- [ ] Las 4 fases tienen duración, participantes, actividades, métricas y criterio de avance
- [ ] Los early adopters están bien elegidos (considerando los perfiles)
- [ ] La estrategia de resistencia es empática y específica para cada persona
- [ ] Fernando (junior entusiasta) tiene alguna medida para evitar dependencia excesiva
- [ ] Los riesgos son realistas y las mitigaciones son concretas
- [ ] El plan total no supera los 3 meses hasta Fase 3
```
